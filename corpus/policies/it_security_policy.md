# IT Security Policy

*Effective: 2026-01-01 · Owner: IT Security · ACME GmbH*

## Scope and Purpose

This policy defines the baseline security rules that apply to all ACME GmbH employees, contractors, and working students who use company systems, devices, or data. It covers authentication, access management, and incident reporting. Violations may lead to access revocation and disciplinary measures. Questions about this policy go to the IT helpdesk (it-helpdesk@acme.de); security incidents go to security@acme.de.

## Password Resets

All password resets are handled via the IT helpdesk with identity verification. Employees who have forgotten a password or suspect it has been compromised must open a helpdesk ticket or call the helpdesk hotline; the helpdesk verifies the requester's identity (for example via a video call with photo ID or a callback to the phone number on file in the HR system) before resetting any credential. Passwords are never communicated by email or chat; the helpdesk issues a one-time temporary password that must be changed at first login. No one at ACME — including IT staff and managers — will ever ask an employee to reveal a password, and such requests must be reported as a security incident.

## Access Requests and Approvals

Access to any internal system beyond the standard onboarding set requires a formal access request. Every access request requires approval from both the requesting employee's manager and the system owner of the target system before the helpdesk provisions access. Requests are submitted through the IT helpdesk portal and must state the business reason and the required permission level; access is granted with the minimum privileges necessary (least privilege). Access rights are reviewed quarterly by system owners, and all access is revoked automatically on the employee's last working day. Sharing personal accounts or credentials with colleagues is prohibited.

## Multi-Factor Authentication

MFA is mandatory on all systems. Every ACME account — including email, VPN, the HR portal, the order-management system, and all cloud services — must be protected with multi-factor authentication using the company-approved authenticator app or a hardware security key. Disabling MFA, even temporarily, requires an exception approved by IT Security and is time-limited. Employees who lose their MFA device must contact the IT helpdesk immediately so the device can be revoked and re-enrolled after identity verification, following the same verification procedure as password resets.

## Phishing and Incident Reporting

Employees must report phishing to security@acme.de. Suspicious emails should be forwarded as attachments without clicking links, opening attachments, or replying to the sender. The same address is used to report any other suspected security incident: lost or stolen devices, suspected malware, unexpected MFA prompts, or possible data leaks. Reports should be made immediately — early reporting is never penalized, even if the employee clicked a link before realizing the message was malicious. IT Security acknowledges incident reports within one business day and coordinates the response; employees must not attempt to investigate or remediate incidents on their own.
